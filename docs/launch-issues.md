# Launch Issue Backlog

Use this list to seed GitHub issues before the announcement. These are intended
to make the project immediately navigable for new contributors.

## Good First Issues

- Try `haitaka_cli play` against Annan and report one confusing engine move.
- Add one Annan SFEN edge case with expected legal/illegal moves.
- Improve `docs/benchmarks.md` with baseline numbers for your machine.
- Add a tiny example package generated by `haitaka_cli package` to release notes.

## Rules And Move Generation

- Build an Annan perft corpus from known Shogitter positions.
- Verify illegal pawn-drop mate handling on more Annan positions.
- Add regression tests for discovered checks involving Annan backing pieces.
- Document the minimum evidence needed to accept a new variant implementation.

## Performance

- Benchmark `legals` before and after an Annan movegen cleanup.
- Investigate move ordering impact on `haitaka_wasm` handcrafted search.
- Add manual benchmark results for common developer machines.

## Search And Evaluation

- Add more tactical SFENs where the handcrafted evaluator makes poor moves.
- Compare fixed-depth self-play at depths 1-4 and report approximate Elo deltas.
- Explore time-based self-play once search time management is more stable.

## NNUE And Training

- Run a small Annan data generation job and post a training report.
- Verify that exported standard shogi NNUE still loads after packaging changes.
- Investigate full shogi move encoding for trainer move fields.

## Shogitter Integration

- Validate the `.tgz` layout expected by `haitaka_cli package`.
- Add Shogitter staging loader support for `haitaka-package.json`.
- Define how packaged engines declare supported rule IDs and model metadata.
